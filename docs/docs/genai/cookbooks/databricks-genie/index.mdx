# Evaluating Databricks Genie Spaces

A complete pipeline for ingesting, evaluating, and improving a Databricks Genie space using MLflow. These cookbooks walk through tracing Genie conversations, running automated and human evaluation, and generating actionable space improvements.

## Pipeline Overview

| Step | Cookbook | What it does |
|------|---------|-------------|
| 1 | [Tracing Pipeline](/genai/cookbooks/databricks-genie/notebooks/tracing-pipeline) | Ingests Genie conversations into Delta with MERGE deduplication, then creates MLflow traces with spans for text-to-SQL, execution, and response generation. |
| 2 | [Evaluation with LLM Judges](/genai/cookbooks/databricks-genie/notebooks/evaluation-judges) | Runs built-in judges (relevance, safety, groundedness), custom LLM judges, and code-based scorers on traces. Flags failures and creates a labeling session for human review. |
| 3 | [Space Improvement Generator](/genai/cookbooks/databricks-genie/notebooks/space-analyzer) | Collects human feedback, extracts the Genie space config, inspects Unity Catalog metadata, and generates structured recommendations (SQL expressions, text instructions, benchmarks) that can be auto-applied. |

:::tip[Prerequisites]
All cookbooks in this series require:
```bash
pip install databricks-sdk mlflow>=3.1.0
```
They run on Databricks and require a Genie space. Start with the Tracing Pipeline, then work through Evaluation and the Space Analyzer.
:::
